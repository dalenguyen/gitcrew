Commands

Everything you need, nothing you don't

Six commands to bootstrap, run, and monitor your agent team. All bash, zero dependencies.

⚙️

gitcrew init

Bootstrap .agent/ in any repo with task board, prompts, test wrapper, roles, and monitoring scripts. One command setup.

🚀

gitcrew spawn

Start an agent loop with any CLI tool. Supports claude, cursor, aider, and codex. Terminal or Docker.

📋

gitcrew task

Manage the shared task board. Add, lock, complete, unlock tasks. Import in bulk. Git conflicts act as a mutex.

📈

gitcrew monitor

Live terminal dashboard showing task status, active branches, recent commits, and the agent log. Auto-refreshes.

🩹

gitcrew doctor

Pre-flight checks for your project: git setup, test infrastructure, role files, CLI tools. Auto-fix with --fix.

🔒

gitcrew hooks

Install a pre-push hook that blocks broken code. Agents (and humans) can't push unless tests pass.

Quickstart

Up and running in 5 minutes

From zero to a parallel agent team on your existing project.

Install gitcrew

One-line install. Requires only bash and git.

curl -fsSL https://raw.githubusercontent.com/dalenguyen/gitcrew/main/install.sh | bash

Bootstrap your repo

Creates the .agent/ directory with all coordination files.

cd your-project/
gitcrew init
gitcrew doctor

Seed the task board

Add real tasks from your issue tracker.

gitcrew task add "Fix: login bug"
gitcrew task add "Feature: CSV export"
gitcrew task list

Spawn agents

Launch 2-3 agents with different roles.

gitcrew spawn Agent-A feature
gitcrew spawn Agent-B bugfix
gitcrew spawn Agent-C quality

Monitor

Watch your team work in a separate terminal.

gitcrew monitor

Scale

Once stable, add more agents or use Docker isolation.

gitcrew spawn Agent-D docs
gitcrew spawn Agent-E feature --docker

Specialization

Built-in agent roles

Each agent gets a role that focuses its behavior. Create custom roles for your team's needs.

🔨

Feature

Picks feature tasks. Writes tests first, then implements.

🐛

Bugfix

Finds failing tests and fixes them with regression tests.

✨

Quality

Refactors without changing behavior. Removes duplication.

📖

Docs

Keeps documentation accurate. Maintains changelog.

🔀

Integration

Merges branches. Gatekeeps main. Reports failures.

Deployment

Three ways to run your team

From chat sessions to fully isolated Docker containers. Start simple, scale up.

Approach Setup Isolation Best For
Chat Sessions Zero Shared filesystem Quick experiments, 2-3 agents
Terminal Loops Minimal Shared filesystem Production use, any scale
Docker Containers Moderate Full isolation Maximum safety, CI/CD

Philosophy

Tests are the source of truth

"Claude will work autonomously to solve whatever problem I give it. So it's important that the task verifier is nearly perfect, otherwise Claude will solve the wrong problem."

— Anthropic Engineering

Your agents are only as good as the test harness you give them. If tests are slow, agents waste time. If tests are verbose, agents lose context. If tests have gaps, agents ship broken code. Invest 80% of your setup time in making tests fast, deterministic, and comprehensive. The agents will figure out the rest.
